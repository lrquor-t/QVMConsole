- 异步任务处理器统一采用 `func(ctx context.Context, task *model.Task, progress func(int, string)) (string, error)` 签名，支持上下文取消与实时进度回调。
- 长耗时操作（如 VM 创建、克隆、迁移）均通过 `taskqueue.RegisterHandler` 注册为异步任务，避免阻塞 HTTP 请求响应。
- 关键后台任务（如带宽重平衡、缓存刷新）在任务完成后通过 `go func()` 异步触发，确保主流程的低延迟响应。
- 模块间依赖通过显式的函数注入（Dependency Injection）方式传递，而非直接 import 具体实现，增强了子包的可测试性与隔离性。

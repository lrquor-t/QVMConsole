# 用户资源管理

## 功能概述

管理员可为每个普通用户设定资源配额（CPU核心数、内存、磁盘、虚拟机数量），用户登录后可在配额范围内自行从模板创建虚拟机。管理员手动分配给用户的虚拟机也计入该用户的配额。

## 角色说明

| 角色 | 权限 |
|------|------|
| 管理员 (admin) | 全部功能，CPU/内存/磁盘/VM 无配额限制，存储配额可配置（默认 0 不限） |
| 普通用户 (user) | 只能看到/操作自己的VM，受全部配额限制 |

普通用户增加云类型：

| 云类型 | 说明 |
|------|------|
| 弹性云 (`elastic`) | 默认类型，继续使用用户级 CPU、内存、磁盘、VM 数量、存储、流量、带宽、端口转发和快照配额 |
| 轻量云 (`lightweight`) | 管理员分配 VM，用户不能新建/编辑/删除 VM，不走用户级配额；流量、带宽、端口转发和快照按单台 VM 配置 |

## 管理员操作

### 创建用户（含配额）

- 用户管理页面 → 新增用户
- 可设定用户名、密码、角色
- 普通用户需设定资源配额：
  - **CPU 核心数**：用户所有VM的CPU总核心数上限
  - **内存 (GB)**：用户所有VM的内存总量上限
  - **磁盘 (GB)**：用户所有VM的磁盘总容量上限
  - **VM 数量**：用户可拥有的虚拟机总数上限
  - **存储 (GB)**：用户「我的存储」空间上限，管理员默认 0（不限），普通用户默认 10
  - **总运行时长 (小时)**：用户所有运行中/暂停中 VM 的累计总运行时长上限
  - **端口转发数量**：用户当前可创建的端口转发规则总数上限，默认 `10`
  - **快照数量**：用户当前可创建的快照总数上限，弹性云默认 `5`
  - **设为 0 表示不限制**

### 修改配额

- 用户管理页面 → 对应用户行 → 点击「配额」按钮
- 可随时调整各项资源配额

### 封禁用户

- 用户管理页面 → 对应用户行 → 点击「封禁」按钮
- 系统会立即使该用户登录失效
- 同时异步关闭该用户下所有运行中的 VM、编排与容器，并关闭 SSH 访问
- **不会删除用户资产**

### 分配虚拟机

- 用户管理页面 → 对应用户行 → 点击「分配VM」按钮
- 管理员可将已有虚拟机分配给用户
- **分配的VM同样计入用户配额**
- 轻量云用户分配 VM 时，需要为每台 VM 设置上下行月流量、上下行带宽、端口转发上限和快照上限；后端会自动绑定该用户的专用 VPC 并创建 VM 专属安全组

## 普通用户操作

### 首页 Dashboard

- 登录后默认跳转到首页
- 显示资源配额概览（CPU/内存/磁盘/VM数量/总运行时长的已用/上限，带进度条）
- 显示端口转发配额的已用/上限
- 显示当前拥有的虚拟机列表概览
- 提供快捷操作入口

### 从模板创建虚拟机

- 虚拟机列表页 → 点击「从模板创建」按钮
- 选择模板、设定VM名称、CPU、内存、磁盘大小
- 界面实时显示剩余配额
- 提交后后端二次校验配额，确保不超限

### 管理自己的虚拟机

- 虚拟机列表仅显示自己拥有的VM
- 可执行：开机、关机、重启、编辑配置、查看详情、删除

## API 接口

### 管理员接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/user/list` | 获取用户列表（含配额信息） |
| POST | `/api/user` | 创建用户（含配额） |
| PUT | `/api/user/:username/quota` | 更新用户配额 |
| PUT | `/api/user/:username/status` | 封禁/解封用户 |
| GET | `/api/user/:username/quota` | 获取用户配额使用情况 |
| PUT | `/api/user/:username/vms` | 分配虚拟机给用户 |
| DELETE | `/api/user/:username` | 删除用户（异步任务，级联删除所有虚拟机和存储池） |

删除用户任务会在删除虚拟机后继续清理该用户的网络资源，包括旧 OVS 静态 IP 绑定、指向该用户 VPC 子网的端口转发、VPC 交换机运行态、交换机 DHCP 配置/租约、VPC VM 绑定、安全组策略和交换机月流量记录。

### 用户自助接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/self/quota` | 查看自己的配额 |
| GET | `/api/self/vms` | 查看自己的VM列表 |
| POST | `/api/self/vm/clone` | 从模板克隆VM |
| DELETE | `/api/self/vm/:name` | 删除自己的VM |

## 配额计算规则

- **磁盘配额**：按虚拟机**配置值**统计（非实际使用量），包含所有VM（无论是否运行）
  - CPU：统计 `virsh dominfo` 中的 `CPU(s)` 字段
  - 内存：统计 `virsh dominfo` 中的 `Max memory` 字段（向下取整到GB）
  - 磁盘：统计主磁盘的 `virtual size`（配置容量）
  - 包含用户自建和管理员分配的所有虚拟机
- **CPU/内存配额**：仅统计**已开机（running）虚拟机**的资源，用户可自由创建和修改配置，但开机时超出配额将被拒绝

## 配额检查时机

| 操作 | 检查内容 |
|------|----------|
| 创建/克隆虚拟机 | 磁盘、VM数量 |
| 创建/克隆并自动开机 | 额外检查总运行时长配额是否已耗尽 |
| 新增磁盘 | 检查新增磁盘大小是否超出磁盘配额 |
| 磁盘扩容 | 计算扩容差值（目标大小 - 当前大小），仅增加时检查 |
| **开机** | **CPU、内存（基于已运行VM统计，超出需先关闭其他VM）+ 总运行时长配额** |

> **注意**：管理员（admin）角色不受配额限制，以上检查仅对普通用户生效。
> 修改CPU/内存配置时不触发配额检查，配额仅在开机时按运行状态校验。

## 总运行时长配额说明

- 统计范围包含 `running` 和 `paused` 状态
- 按用户所有 VM 的总和累计
  - 例如 2 台 VM 同时运行 1 小时，会累计 2 小时
- 当剩余时间不足 2 小时时，系统会自动发送邮件提醒
- 当配额耗尽时，系统会自动关闭该用户当前运行中的所有 VM，并禁止继续开机
